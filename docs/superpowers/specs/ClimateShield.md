# 🛡️ ClimateShield（气候盾牌）设计文档

> 为 One4Earth Climate Change-Makers Challenge 2026 黑客松设计

---

## 产品定位

**名称：** ClimateShield（气候盾牌）

**一句话：** 帮社区志愿者为脆弱邻居生成"救命级"的离线备灾物料。

**核心洞察：** 极端天气预警系统已经很先进了——但它们全部依赖智能手机和网络。而每次气候灾害中最先死亡的，恰恰是不会用手机的独居老人、残障人士和语言不通的新移民。**信息的"最后一公里"断裂，正在杀人。**

**Slogan：** *"The warning was on an app. They didn't have the app."*

---

## 核心设计哲学

### ❗ 关键翻转：谁是用户？

| | 传统思路 | ClimateShield |
|---|---|---|
| **App 给谁用** | 给所有人（包括老人） | 给**帮助老人的那个年轻人** |
| **输出物** | 屏幕上的推送通知 | **可打印的纸质物料** |
| **假设前提** | 每个人都有智能手机 | 承认数字鸿沟的存在 |

> **设计核心：数字工具产出模拟世界的救命物资。**
> App 是桥梁，纸是终端。

## ⭐ 四个视角（Four Lens）的深度体现

这是本项目的**最大竞争力**——四个视角不是硬凑的，而是设计的核心。

### 1. Equity & Justice Lens（公平与正义）

> *"In 2021, a heat dome killed 619 people in British Columbia. Most were elderly, living alone."*

- 专为**最脆弱群体**设计：独居老人、残障人士、语言不通的移民
- 他们**不需要会用任何 App**——输出物是纸质的、大字的、贴在门上的
- 多语言支持：同一物料一键切换语言
- 免费、开源，任何社区都能用

### 2. Practical Lens（实操性）

- 输出是**纸**——全世界最实用、最抗断电、最抗断网的信息介质
- 不需要下载 App、不需要注册账号、不需要网络
- 一个志愿者 5 分钟就能为整栋楼生成定制物料
- MVP 极简，48 小时完全可交付

### 3. Two-Eyed Seeing Approach（双眼观察 / 原住民知识整合）

- 备灾建议中整合**传统社区智慧**：
  - 原住民世代传承的灾害预判方法（读风向、看水位、观察动物行为）
  - 中国传统的邻里守望互助文化
  - 各地社区口口相传的应急经验
- 与**现代气候科学数据**并列展示，而非替代
- 报告中设有"社区智慧"板块，鼓励志愿者补充本地传统知识

### 4. Systems Approach（系统思维）

```
志愿者/居委会（数字端）
    ↓ 输入社区位置
AI 风险评估引擎
    ↓ 生成定制化内容
可打印物料包（物理端）
    ↓ 分发到户
脆弱群体获得信息
    ↓ 提前准备
社区整体韧性 ↑↑↑
    ↓ 灾后反馈
更新下一轮物料
```

完整闭环：**数字 → 物理 → 人 → 社区 → 反馈 → 迭代**

## 输出物料详细设计

### 物料 1：大字版应急清单

```
┌─────────────────────────────────────┐
│  ⚠️  暴雨来了，请这样做：            │
│                                     │
│  ✅  关好门窗                        │
│  ✅  准备3天的饮用水                  │
│  ✅  手电筒放在床头                   │
│  ✅  手机充满电                       │
│  ✅  不要走地下通道                   │
│                                     │
│  📞 紧急电话：___________            │
│  📞 邻居电话：___________            │
│                                     │
│  🛡️ ClimateShield | 你的社区在守护你  │
└─────────────────────────────────────┘
```

**设计原则：**
- 字号 ≥ 24pt（老花眼可读）
- 纯图标 + 短句（零认知负荷）
- 高对比度（黑底白字 / 白底黑字）
- 多语言版本一键切换

### 物料 2：门贴卡片

```
┌──────────────────────────┐
│                          │
│    🔴 我需要帮助          │
│    I NEED HELP           │
│                          │
│  □ 我是独居老人           │
│  □ 我行动不便             │
│  □ 我需要用药             │
│  □ 我有宠物需要照顾       │
│                          │
│  紧急联系人：_________    │
│                          │
│  🛡️ ClimateShield        │
└──────────────────────────┘
```

**用法：** 住户自愿贴在门上内侧。灾害发生时翻到外侧，救援者/邻居一目了然。

### 物料 3：社区避难地图

- 基于输入位置自动生成的简化地图
- 标注：最近避难所、供水点、医疗点、高地（防洪）
- 适合打印为 A3 海报，张贴在电梯/楼道/公告栏

## 技术架构

### 技术选型（极简原则）

| 层级 | 技术 | 理由 |
|------|------|------|
| **前端** | Next.js / 纯 HTML+JS | 单页应用，快速开发 |
| **UI 框架** | TailwindCSS + 手写组件 | 确保打印适配 |
| **AI 层** | DeepSeek / GPT API | 根据位置生成定制化备灾建议 |
| **地图** | Mapbox / Leaflet | 生成社区避难地图 |
| **PDF 生成** | html2canvas + jsPDF 或 @react-pdf/renderer | 生成可打印物料 |
| **气候数据** | Open-Meteo API / 公开气候风险数据 | 获取区域气候风险信息 |
| **部署** | Vercel / Cloudflare Pages | 免费、快速 |

### 架构图

```
用户输入地址
    │
    ├──→ 地理编码 API（地址 → 坐标）
    │
    ├──→ 气候风险数据 API（坐标 → 风险类型+等级）
    │
    ├──→ AI（LLM）生成定制化建议文案
    │        ├── 应急清单（基于风险类型）
    │        ├── 避难建议
    │        └── 多语言翻译
    │
    └──→ 前端渲染
             ├── 屏幕预览
             └── PDF 导出（打印优化）
```

**⚠️ 刻意不做的事：**
- ❌ 用户注册/登录系统
- ❌ 数据库
- ❌ 后端服务器
- ❌ 社交功能
- ❌ 实时预警推送

**理由：** 这些功能会分散48小时的精力，且不影响核心 Demo 效果。MVP 就是一个精美的单页工具。

## 竞品差异化

| 对比对象 | ClimateShield 的差异 |
|---------|---------------------|
| 政府预警系统 | 我们解决"最后一公里"——到达不会用手机的人 |
| 红十字会备灾指南 | 我们根据**具体位置**定制化，而非通用手册 |
| 社区 App（Nextdoor 等） | 我们的输出是**纸质的**，不排斥任何人 |
| 现有备灾工具 | 我们专注于**脆弱群体**，而非一般居民 |

---

## 风险与应对

| 风险 | 应对方案 |
|------|---------|
| AI 生成的建议不够准确 | 预设高质量模板 + AI 仅做定制化微调 |
| 气候风险数据源难获取 | 使用 Open-Meteo 历史数据 + 公开的自然灾害数据库 |
| PDF 排版在不同设备上不一致 | 使用固定尺寸的 Canvas 渲染，确保一致性 |
| 48小时时间紧 | MVP 就做：一个地址输入 → 一套物料输出，砍掉一切非核心功能 |

## 为什么这个项目会赢

1. **真实的痛点：** 不是想象出来的问题，是每年都在杀人的信息鸿沟
2. **优雅的设计翻转：** "App 不是给老人用的"这个洞察本身就足以打动评委
3. **完美契合 Four Lens：** 不是硬凑，四个视角就是设计的骨架
4. **全球通用：** 不绑定任何国家，任何社区都能用
5. **48小时可交付：** 没有后端、没有数据库、没有用户系统，集中火力做一件事做到极致
6. **Pitch 有故事：** BC热穹顶619人死亡——这是加拿大评委自己的记忆
