# 📑 项目 PRD：Urban Miner (城市矿工)
**定位：** AI 驱动的电子垃圾资产化与逆向物流撮合平台
**Slogan：** Don't trash it, Mine it. (让每一件旧物，成为城市的宝藏)

---

### 1. 产品定位 (Product Positioning)
* **核心矛盾：** 电子垃圾（E-waste）回收的“第一公里”成本过高。个人用户懒得处理，专业机构收不到货。
* **解决方案：** 利用 **AI Vision (计算机视觉)** 瞬间完成废旧电子产品的“识别+估值”，将负资产转化为可交易的“悬赏包（Bounty）”，利用共享经济模型完成物资流转。
* **目标用户：** * **C 端：** 拥有闲置电子产品、追求极简生活或额外收入的普通人。
    * **B 端：** 独立回收员、极客玩家、小型环保初创企业。

---

### 2. 核心用户路径 (User Flow)

#### C 端：寻宝者路径 (The Finder)
1.  **AI 扫描：** 用户打开 PWA，调用摄像头对准一堆电子垃圾（手机、线缆、主板）。
2.  **资产鉴定：** UI 以“呼吸感”特效展示识别过程，显示预估含金量、含铜量及**实时市场回收底价**。
3.  **发布矿包：** 用户确认信息，一键生成带有地理位置信息的“矿包（Bounty）”。
4.  **收益入账：** 回收员上门并扫码确认后，资金即时到账。

#### B 端：矿工路径 (The Scrapper)
1.  **资源热力图：** 地图展示附近待领取的“矿包”分布及预估价值总额。
2.  **智能接单：** 系统根据路线最优原则推荐多个邻近矿包，一键打包接单。
3.  **线下核销：** 导航至目的地，扫码验证矿包，完成物理交割。

---

### 3. 技术架构 (Technical Architecture)
由于 JohnLin 擅长高性能后端和基建，架构设计如下：

* **前端 (Frontend):** * React / Next.js 构建的 PWA（确保跨平台且轻量）。
    * **交互重点：** 视觉识别时的动态反馈、地图组件集成。
* **AI 推理层 (AI Inference):** * **GPT-4V / Claude 3.5 Sonnet API：** 负责多模态图像识别。
    * **LLM 结构化输出：** 提取物种、型号，并从模拟的大宗商品数据库中匹配实时单价。
* **后端基建 (Backend):** * **Golang / Rust 微服务：** 负责高并发的订单撮合与位置计算。
    * **PostgreSQL + PostGIS：** 处理地理位置索引，实现热力图功能。
* **API Gateway:** 统一管理 C/B 两端的请求路由。

### 5. 差异化总结 (Competitive Advantage)
* **从“情感动员”转向“经济驱动”：** 相比其他环保项目，我们直接给用户算账，用钱驱动行为，落地性极强。
* **UI/UX 的维度打击：** 利用视传背景，将枯燥的“收破烂”包装成极具未来感的“城市寻宝”，改变用户的心理模型。
* **硬核基建支撑：** 并非简单的表单提交，而是基于 AI 视觉识别和动态位置撮合的硬核系统，展示了团队处理复杂业务的能力。
