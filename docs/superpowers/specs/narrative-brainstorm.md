# Eco Evidence — 叙事头脑风暴

> 团队内部讨论文档 | 2026-04-04

---

## Part 1：为什么"对接环保组织"的链条是断裂的

### 当前叙事链条

```
居民拍照 → AI识别物种 → 聚合数据 → 生成报告 → 发布到平台 → 环保组织看到 → 展开合作 → 保护环境
                                                    ↑                    ↑
                                               这里开始断裂          这里完全断裂
```

### 为什么不成立

| 假设 | 现实 |
|------|------|
| "环保组织会来看我们的平台" | 环保组织连自己的邮件都看不完，不会主动来一个新平台找数据 |
| "有了数据他们就会行动" | 环保组织**不缺数据**——IUCN红色名录、eBird、iNaturalist 已有数十亿条记录。他们缺的是**资金、政策支持和人力** |
| "我们的报告能推动合作" | 一份学生生成的 PDF 在专业环保组织眼里没有权威性。他们信赖的是同行评审论文和政府监测数据 |
| "这是一个有效的闭环" | 从"一个人拍了一张鸟的照片"到"一个环保组织决定采取行动"之间，至少缺少5个中间步骤，每一步的转化率都接近于零 |

### 核心问题

**你不能在 4 分钟 Pitch 里让评委相信"有人拍照片就能让环保组织行动起来"。** 

评委会问的第一个问题一定是：**"你跟哪个环保组织聊过？他们真的需要这个吗？"**

如果答不上来，整个项目的可信度就崩了。

## Part 2：同样的问题也存在于"学校委员会"叙事

```
学生观察 → 生成报告 → 递交学校委员会 → 委员会决定保护湿地
                                          ↑
                                    这里也不太可信
```

学校委员会管的是预算、课程和人事，**土地规划不归他们管**。如果是市政府要修停车场，一所小学的学生报告不太可能改变决策。

**所以：不要把故事的"成功"定义为"改变了政策"。**

---

## Part 3：换一个思路——什么样的叙事链条是完整的？

关键原则：**故事的价值不需要依赖外部行动者的响应。**

也就是说：别让故事成功与否取决于"别人看到报告后会不会做什么"。
让**生成报告这个行为本身**就是有价值的。

以下是几条完整的叙事链：

---

### 📖 叙事 A：教育工具（最安全、最可信）

**故事：** 一位自然科学老师用 Eco Evidence 作为教学工具，让学生亲手记录校园周围的生物多样性，学期末生成一份报告作为课程成果。

**价值闭环：**
```
学生观察 → 记录数据 → 生成报告 → 学到了科学方法 ✓
                                   （完，不需要任何人响应）
```

**优点：**
- 链条 100% 闭合——学到东西就是价值
- 对标 One4Earth 的教育基因（他们本身就做 youth engagement）
- 学生作为行动者 = 极强的 Equity 叙事

**缺点：**
- 不够"热血"，少了一点"改变世界"的感觉
- 可能显得格局不够大

**Pitch 结尾可以说：**
> "Today it's a school project. Tomorrow, when these students grow up, they'll have the data literacy to fight for their communities."

---

### 🏦 叙事 B：生态证据银行（最有深度）

**故事：** 一个社区的居民持续记录本地生态。他们不是为了"现在"改变什么——他们是在**存证据**。当未来某天开发商提出在这里建项目时，他们已经有了 3 年的生态基线数据。

**价值闭环：**
```
居民持续观察 → 数据逐年积累 → 形成基线 → 未来某天需要时，证据已经在手 ✓
                                            （价值在"准备"，不在"行动"）
```

**优点：**
- 逻辑非常强——"平时不急，急时有据"
- 契合长期主义，符合 Systems Approach
- 类比保险：你买保险不是因为你今天要出事

**缺点：**
- 演示时比较抽象——你不能展示"三年后的效果"

**Pitch 核心台词：**
> "You don't wait until the fire to buy a fire extinguisher. Eco Evidence helps communities build their ecological evidence bank — so when developers come knocking, the data is already there."

---

### 🪶 叙事 C：原住民土地守护（最契合本赛事）

**故事：** 一个加拿大原住民社区用 Eco Evidence 记录其传统领地上的生态变化。长老们分享记忆中的物种（"我小时候这条河里满是三文鱼"），年轻人用 App 记录现在看到的。报告将两者并列展示。

**价值闭环：**
```
长老口述过去 + 年轻人记录现在 → 生成"过去 vs 现在"的对比报告 ✓
                                  （代际知识传承本身就是价值）
```

**优点：**
- **完美命中 Two-Eyed Seeing**——这正是他们最强调的评分视角
- 原住民+土地+生物多样性，赛事组织方 One4Earth 最关心的三件事全中
- 情感冲击力最强

**缺点：**
- 你们不是原住民，讲这个故事要小心不越界
- 不要代表原住民说话，而是说"我们为他们构建工具"

**Pitch 核心台词：**
> "Elders remember the salmon. Young people see empty rivers. Eco Evidence bridges those two ways of knowing — so no community's ecological memory is ever lost."

---

### 🌡️ 叙事 D：气候变化的"个人证据"（最有情感）

**故事：** 你的爷爷说"我小时候每年夏天都能看到萤火虫"。你的孩子从没见过萤火虫。这不是偶然——这是气候变化。但你怎么证明？

Eco Evidence 让每个人记录自己身边的自然变化。一个人的观察是故事，一万个人的观察是证据。

**价值闭环：**
```
个人记录身边的自然 → 聚合成社区/城市级数据 → 可视化变化趋势 ✓
                                               （让气候变化变得个人化、可感知）
```

**优点：**
- 情感共鸣极强——每个人都有类似的记忆
- "个人故事 + 集体数据" = Collective Action 的最佳诠释
- 全球通用，不绑定任何国家

**缺点：**
- 需要长时间的数据积累才有说服力
- Demo 中需要用模拟数据来展示"几年的变化"

**Pitch 开场：**
> "My grandmother remembers catching fireflies every summer evening. My daughter has never seen one. Climate change isn't just in the headlines — it's in our backyards. But how do you prove what's no longer there?"

## Part 4：我的推荐排序

| 排名 | 叙事 | 理由 |
|------|------|------|
| 🥇 | **C 原住民土地守护** 或 **D 气候变化个人证据** | 情感最强 + 赛事契合最高，但需要编排好避免越界 |
| 🥈 | **B 生态证据银行** | 逻辑最完整，最不怕评委追问，但演示稍抽象 |
| 🥉 | **A 教育工具** | 最安全，但缺乏 wow factor |

### 混合建议

实际上可以**混合使用**——比如核心叙事用 D（萤火虫的情感开场），Demo 场景用 A（学校自然课），Future Vision 用 B（长期证据银行化），Two-Eyed Seeing 用 C（长老记忆板块）。

**最终的 Pitch 结构可以是：**
```
情感开场（D：爷爷的萤火虫）
    ↓
问题陈述（有观察无报告）
    ↓
解决方案 + Demo（A：学校场景演示核心功能）
    ↓
Four Lens 展开（C：原住民知识 + B：长期证据银行）
    ↓
收尾升华（D：让每个人的观察都成为证据）
```
