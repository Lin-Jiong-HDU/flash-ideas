# Eco Evidence（生态证据）设计文档

> 为 One4Earth Climate Change-Makers Challenge 2026 黑客松设计

---

## 产品定位

**名称：** Eco Evidence（生态证据）

**一句话：** 把社区生物多样性观察转化为可行动的倡导报告。

**核心洞察：** 现有的公民科学App（iNaturalist等）解决了"识别物种"的问题，但缺失把数据变成社区可以用来倡导改变的武器。

**目标用户：**
- 乡村/郊区学校的学生和教师
- 社区环保组织
- 想要保护本地生态的居民

**核心价值：** 现有工具帮你"认识物种"，我们帮你"用数据说话"。

## 四个视角的体现

| 视角 | 如何体现 |
|------|----------|
| **Equity & Justice Lens** | 开源免费，面向资源较少的乡村社区；学生作为变革者；关注边缘化社区的声音 |
| **Practical Lens** | MVP 极简，4天可完成；调用现成AI不重复造轮子；离线可用 |
| **Two-Eyed Seeing Approach** | 报告中可整合"以前 vs 现在"的对比，鼓励采访长辈记录传统知识 |
| **Systems Approach** | 数据 → 可视化 → 报告 → 政策，形成系统闭环；不仅是观察工具，更是倡导工具 |

## 核心功能流程

```
┌──────────────────────────────────────────────────────────────┐
│                        Eco Evidence                          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  STEP 1: 记录                                     │
│  ├── 拍照或从相册导入                                         │
│  ├── AI 识别物种（调用 iNaturalist API）                      │
│  └── 自动记录：时间 + 地点 + 物种                              │
│                                                              │
│  STEP 2: 聚合                                              │
│  ├── 选择/创建观察点                                          │
│  └── 同一地点的记录自动归类                                    │
│                                                              │
│  STEP 3: 可视化                                         │
│  ├── 时间线：物种出现/消失                                    │
│  ├── 物种数量变化曲线                                         │
│  └── 物种多样性指数趋势                                        │
│                                                              │
│  STEP 4: 报告生成 ← 核心差异化                     │
│  ├── 自动生成社区倡导报告                                      │
│  ├── 包含：数据 + 图表 + 建议措施                              │
│  └── 导出 PDF / 分享链接 / 给环保组织看                                      │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 报告智能分析

| 自动分析 | 说明 |
|----------|------|
| **下降警报** | 某物种连续 X 周未出现 → 标记为"可能消失" |
| **新增发现** | 首次记录的物种 → 标记为"新发现" |
| **季节性建议** | 根据季节推荐适合的本土植物种植 |
| **行动模板** | 根据物种类型（鸟类/昆虫/植物）匹配不同建议 |

## 技术架构

### 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| **前端框架** | Next.js 14 (App Router) | 全栈框架，前后端一体化，部署简单 |
| **UI** | TailwindCSS + shadcn/ui | 高质量组件，快速搭建专业 UI |
| **图表** | Recharts | React 生态，简单易用 |
| **后端** | Next.js API Routes | 与前端一体化 |
| **数据库** | Supabase (PostgreSQL) | 免费 tier 足够，Auth + DB + Storage 一站式 |
| **文件存储** | Supabase Storage | 与数据库集成 |
| **物种识别** | iNaturalist API | 免费开源，识别准确率高 |
| **报告生成** | @react-pdf/renderer | React 组件直接生成 PDF |

### 数据库设计

```sql
-- 观察点
CREATE TABLE locations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  description TEXT,
  coordinates POINT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  owner_id UUID REFERENCES auth.users(id)
);

-- 观察记录
CREATE TABLE observations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  location_id UUID REFERENCES locations(id),
  species_name TEXT NOT NULL,
  species_common_name TEXT,
  confidence FLOAT,
  photo_url TEXT,
  observed_at TIMESTAMP DEFAULT NOW(),
  observer_name TEXT,
  notes TEXT
);

-- 报告
CREATE TABLE reports (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  location_id UUID REFERENCES locations(id),
  date_range_start DATE,
  date_range_end DATE,
  generated_at TIMESTAMP DEFAULT NOW(),
  pdf_url TEXT
);
```

### API 端点

```
POST   /api/observations        # 创建观察记录
GET    /api/observations        # 获取观察列表（支持按 location 筛选）
GET    /api/observations/:id    # 获取单条记录

POST   /api/locations           # 创建观察点
GET    /api/locations           # 获取观察点列表
GET    /api/locations/:id       # 获取单个观察点详情

POST   /api/identify            # 调用 iNaturalist API 识别物种
POST   /api/reports/generate    # 生成报告 PDF
GET    /api/reports/:id         # 获取报告
```

## 差异化总结

| 竞品 | Eco Evidence 的优势 |
|------|---------------------|
| iNaturalist | 我们专注"报告生成"而非"物种识别" |
| 传统监测工具 | 我们面向普通社区，不是专业科研人员 |
| 手动报告 | 我们自动化数据收集和报告生成 |
